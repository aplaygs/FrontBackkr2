# Управление Сетевыми Узлами

Информационная система управления активами с разделением прав доступа.

## 🚀 Быстрый запуск

1. **Установите зависимости**:
   ```bash
   npm install
   ```
2. **Запустите сервер**:
   ```bash
   npm run dev
   ```
3. **Откройте приложение**:
   - Перейдите в браузере по адресу: `http://localhost:3000`

---

## 📝 Инструкция по тестированию проекта (Практические задания)

Ниже приведена инструкция на русском языке, которая поможет вам протестировать выполнение заданий 7–11 вашего проекта.

---

### 📓 Тетрадь №1: Задание 7 — Проверка регистрации и хеширования
- **Действие**: Перейдите на страницу «Регистрация». Введите данные нового пользователя.
- **Что проверить**: После нажатия «Регистрация Узла» вы должны быть перенаправлены на страницу логина. В базе данных сервера пароль сохранится в виде зашифрованного хеша (библиотека `bcrypt`).

---

### 📓 Тетрадь №2: Задание 8 — Аутентификация и JWT (Вход в систему)
- **Действие**: Войдите под только что созданным пользователем или под админом (**admin@example.com** / **admin**).
- **Что проверить**: При успешном входе сервер выдает `accessToken` и `refreshToken`. Клиент сохраняет их и отображает ваше имя в навигационной панели. Если ввести неверный пароль, система должна выдать ошибку «Неверные учетные данные».

---

### 📓 Тетрадь №3: Задание 9 — Ролевая модель (RBAC)
- **Действие**:
  1. Зайдите как обычный **Пользователь**. Вы должны видеть список товаров, но кнопки «Добавить Юнит», «Редактировать» или «Удалить» должны быть скрыты.
  2. Зайдите как **Администратор**. Вам должен быть доступен раздел «Админ-панель» и все функции управления товарами.
- **Что проверить**: Ограничение доступа на уровне интерфейса и на уровне API (сервер проверяет роль в токене).

---

### 📓 Тетрадь №4: Задание 10 — Управление пользователями (Только для Админа)
- **Действие**: Перейдите в «Админ-панель».
- **Что проверить**:
  1. Попробуйте изменить роль другого пользователя (например, с «Пользователь» на «Продавец»).
  2. Попробуйте удалить пользователя (кнопка корзины). *Примечание: основного админа удалить нельзя.*

---

### 📓 Тетрадь №5: Задание 11 — Управление каталогом товаров (CRUD)
- **Действие**: Перейдите в раздел «Товары».
- **Что проверить**:
  1. **Добавление**: Нажмите «Добавить Юнит», заполните форму. Товар должен появиться в списке.
  2. **Редактирование**: Нажмите на иконку карандаша у товара, измените цену или описание.
  3. **Удаление**: Нажмите на иконку корзины (доступно только админу). Товар должен исчезнуть.

---

## 🛠 Технологический стек
- **Frontend**: React + Vite + Tailwind CSS + Framer Motion
- **Backend**: Express (Node.js)
- **Безопасность**: JWT + Cookies + Bcrypt (шифрование паролей)
- **Иконки**: Lucide React
